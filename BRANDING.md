# 🎯 Weekly Impact Deck - Project Branding

## Project Name

**Weekly Impact Deck**

## Tagline

> Transform technical work into stakeholder-ready presentations

## Repository Name

`weekly-impact-deck`

## One-Sentence Description

A professional system for tracking, documenting, and presenting weekly software
development updates to non-technical stakeholders.

## Elevator Pitch (30 seconds)

Weekly Impact Deck helps development teams communicate their work to business
stakeholders in language they understand. Instead of technical jargon, it
focuses on business impact—turning "optimized database queries" into "faster
customer experience." It's open source, lightweight, and requires no external
dependencies.

## Key Value Propositions

1. **Business Language First** - Transform technical details into
   stakeholder-friendly impact statements
2. **Automated Workflow** - PowerShell scripts handle setup and statistics
   generation
3. **Professional Presentations** - 12-slide template ready for leadership
   meetings
4. **Developer Friendly** - Markdown-based, Git-compatible, no complex tools
5. **Zero Dependencies** - Pure PowerShell, Markdown, and HTML/CSS/JS

## FAQ Responses

**Q: Why not use Jira?**  
A: Jira is great for task tracking, but this focuses purely on stakeholder
communication. It's complementary, not competitive.

**Q: Does it require a server?**  
A: No! It's just files. Works anywhere PowerShell runs—your laptop, Git repo, or
even a USB drive.

**Q: Can non-technical people use it?**  
A: The PowerShell scripts make setup easy. If you can run a script, you're good
to go. The output is just HTML and markdown.

**Q: How do I customize it?**  
A: Everything is plain text files. Edit the templates, modify the scripts,
change the HTML/CSS. It's yours to adapt.
